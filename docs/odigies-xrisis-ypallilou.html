Field Operations Manual

Οδηγίες Χρήσης για Υπάλληλο

Το παρόν εγχειρίδιο εξηγεί βήμα βήμα πώς χρησιμοποιείται η εφαρμογή διαχείρισης εργασιών πεδίου. Είναι γραμμένο για καθημερινή χρήση από υπάλληλο ή συνεργάτη που πρέπει να βλέπει εργασίες, να τις αναλαμβάνει, να τις ενημερώνει και να τις προωθεί σωστά μέσα στο workflow.

Ρόλοι συστήματος Admin 1, Συνεργάτης 1, Συνεργάτης 2
Βασική λειτουργία Διαχείριση και παρακολούθηση τεχνικών εργασιών πεδίου
Workflow Ανατέθηκε → Προγραμματισμένη → Σε εξέλιξη → Για επικύρωση → Ολοκληρωμένη
Σημαντικό Η εφαρμογή είναι prototype χωρίς database. Τα δεδομένα αποθηκεύονται τοπικά στον browser του χρήστη. Αυτό σημαίνει ότι χρησιμοποιείται για δοκιμή, παρουσίαση, έλεγχο ροής και όχι για τελική παραγωγική χρήση.

1. Τι κάνει η εφαρμογή

Η εφαρμογή χρησιμοποιείται για να οργανώνει τεχνικές εργασίες πεδίου. Κάθε εργασία έχει στοιχεία έργου, διεύθυνση, υπεύθυνο συνεργάτη, προγραμματισμό, φωτογραφίες, αρχεία, υλικά, ιστορικό και τεχνικά metadata.

Ο βασικός στόχος είναι να μην υπάρχει μόνο μια απλή λίστα εργασιών, αλλά πλήρης έλεγχος του κύκλου ζωής κάθε εργασίας από τη δημιουργία μέχρι την έγκριση και το κλείσιμο.

2. Ποιοι ρόλοι υπάρχουν

Ρόλος Τι μπορεί να κάνει
Admin 1 Βλέπει όλες τις εργασίες, δημιουργεί νέες, ορίζει ποιοι συνεργάτες έχουν πρόσβαση, κάνει αλλαγές, εγκρίνει ή απορρίπτει.
Συνεργάτης 1 Βλέπει μόνο τις εργασίες που του επιτρέπει ο admin ή όσες έχει ήδη αναλάβει ο ίδιος.
Συνεργάτης 2 Έχει την ίδια λογική με τον Συνεργάτη 1 και δουλεύει μόνο μέσα στο δικό του επιτρεπόμενο scope.
Η επιλογή ρόλου και χρήστη γίνεται από το πάνω δεξιά μέρος της εφαρμογής. Δεν υπάρχει login με κωδικό στο prototype.

3. Κεντρικές ενότητες της εφαρμογής

Dashboard Δείχνει πόσες εργασίες υπάρχουν σε κάθε κατάσταση.
Εργασίες Δείχνει τον πίνακα με όλες τις ορατές εργασίες και επιτρέπει αναζήτηση και φιλτράρισμα.
Task Detail Είναι η πλήρης καρτέλα της εργασίας με tabs, workflow actions και ιστορικό.
Export PDF Εξάγει αναφορά με όλες τις ανοιχτές εργασίες που μπορεί να δει ο τρέχων χρήστης.

4. Πώς ξεκινά ο χρήστης

1Επιλογή ρόλου

Από το επάνω menu επιλέγουμε αν θα δουλέψουμε ως Admin ή ως Partner.

2Επιλογή χρήστη

Από το δεύτερο dropdown επιλέγουμε συγκεκριμένο χρήστη, π.χ. Admin 1 ή Συνεργάτης 1.

3Έλεγχος dashboard

Στο dashboard βλέπουμε αμέσως πόσες εργασίες υπάρχουν ανά στάδιο για τον τρέχοντα χρήστη.

5. Πώς δημιουργεί εργασία ο Admin

  1. Ο admin πατά το κουμπί Νέα εργασία.
  2. Ανοίγει φόρμα δημιουργίας με τα βασικά πεδία.
  3. Συμπληρώνει τίτλο, είδος, project, project ID, SR / BID, team, διεύθυνση, πόλη και σημειώσεις.
  4. Ορίζει προγραμματισμό αν υπάρχει ήδη γνωστή ημερομηνία και ώρα.
  5. Επιλέγει ποιοι συνεργάτες επιτρέπεται να δουν την εργασία.
  6. Πατά Δημιουργία.

Μετά τη δημιουργία, το σύστημα παράγει αυτόματα:

  • Task ID
  • Created at / Created by
  • Updated at / Updated by
  • Αρχική εγγραφή στο History
  • Κενά πεδία για φωτογραφίες, αρχεία και υλικά

6. Τι σημαίνει κάθε βασικό πεδίο

Πεδίο Επεξήγηση
Τίτλος Το βασικό όνομα της εργασίας.
Είδος Αυτοψία, εγκατάσταση ή επισκευή.
Project Το έργο στο οποίο ανήκει η εργασία.
Project ID Ο κωδικός αναφοράς του έργου.
SR / BID Εσωτερικός ή εξωτερικός κωδικός ticket / αιτήματος.
Πόρος / Team Η ομάδα ή το συνεργείο που σχετίζεται με την εργασία.
Επιτρεπόμενοι partners Ποιοι συνεργάτες θα μπορούν να δουν και να αναλάβουν την εργασία.
Αναλήφθηκε από partner Ποιος συνεργάτης τελικά την έχει αναλάβει.
Ημ/νία έναρξης Πότε έχει προγραμματιστεί να ξεκινήσει.
Ημ/νία λήξης Πότε ολοκληρώθηκε ή αναμένεται να ολοκληρωθεί.
Σημειώσεις Ελεύθερο πεδίο με οδηγίες, παρατηρήσεις ή ενημερώσεις.

7. Πώς βλέπει και αναλαμβάνει εργασία ο συνεργάτης

  1. Ο χρήστης επιλέγει ρόλο Partner.
  2. Επιλέγει τον εαυτό του, π.χ. Συνεργάτης 1.
  3. Πηγαίνει στη σελίδα Εργασίες.
  4. Βλέπει μόνο τις εργασίες που επιτρέπεται να δει.
  5. Πατά πάνω στη γραμμή της εργασίας για να μπει στην καρτέλα της.
  6. Αν η εργασία είναι διαθέσιμη, πατά Ανάληψη εργασίας.
Αν ο συνεργάτης δεν είναι μέσα στους επιτρεπόμενους χρήστες, δεν θα βλέπει την εργασία καθόλου.

8. Πώς προχωρά το workflow

Κατάσταση Τι σημαίνει Ποιος τη μετακινεί
Ανατέθηκε Η εργασία υπάρχει αλλά δεν έχει ολοκληρωθεί η ενεργή εκτέλεση. Admin δημιουργεί την εργασία.
Προγραμματισμένη Υπάρχει συνεργάτης και χρονικό παράθυρο. Admin ή συνεργάτης μετά την ανάληψη.
Σε εξέλιξη Η εργασία εκτελείται στο πεδίο. Ο συνεργάτης πατά Έναρξη εργασίας.
Για επικύρωση Η εκτέλεση ολοκληρώθηκε και αναμένει έλεγχο από admin. Ο συνεργάτης πατά Αποστολή για επικύρωση.
Ολοκληρωμένη Η εργασία εγκρίθηκε και έκλεισε. Ο admin πατά Έγκριση ολοκλήρωσης.

Αν ο admin διαπιστώσει πρόβλημα, μπορεί να πατήσει Απόρριψη και επιστροφή. Τότε η εργασία επιστρέφει σε κατάσταση Σε εξέλιξη.

9. Τι κάνει ο χρήστης μέσα στην καρτέλα εργασίας

Tab Χρήση
Κύριος Τα βασικά στοιχεία της εργασίας, το planning, οι συνεργάτες και οι σημειώσεις.
Φωτογραφίες Ανέβασμα φωτογραφιών σε κατηγορίες πριν, μετά, εξοπλισμός, καλωδίωση.
Αρχεία Ανέβασμα PDF και λοιπών υποστηρικτικών αρχείων.
Υλικά Καταχώρηση υλικών που χρησιμοποιήθηκαν.
Floors Πληροφορίες για ορόφους, μονάδες και risers του κτιρίου.
Health & Safety Checklist ασφάλειας, παρατηρήσεις και σημεία προσοχής.
Ιστορία Δείχνει ποιος έκανε τι και πότε.
Σύστημα Τεχνικά metadata όπως task ID, timestamps και internal flags.

10. Πώς ανεβάζει ο συνεργάτης τεκμηρίωση

  1. Μπαίνει στο tab Φωτογραφίες και επιλέγει την κατάλληλη κατηγορία.
  2. Ανεβάζει όσες φωτογραφίες χρειάζονται.
  3. Μπαίνει στο tab Αρχεία και ανεβάζει report ή PDF.
  4. Αν χρησιμοποιήθηκαν υλικά, τα περνά στο tab Υλικά.
  5. Συμπληρώνει τυχόν παρατηρήσεις στο Health & Safety.

11. Τι πρέπει να κάνει ο Admin πριν εγκρίνει

  • Να ελέγξει αν η εργασία είναι όντως στη φάση Για επικύρωση.
  • Να επιβεβαιώσει ότι υπάρχουν σημειώσεις και σωστό planning.
  • Να ελέγξει φωτογραφίες και αρχεία.
  • Να ελέγξει ότι η καρτέλα δεν έχει εμφανή ελλείμματα.
  • Να διαβάσει το ιστορικό για να δει τι αλλαγές έγιναν.

Αν όλα είναι σωστά, ο admin πατά Έγκριση ολοκλήρωσης. Αν όχι, γράφει σχόλιο και πατά Απόρριψη και επιστροφή.

12. Πώς γίνεται το Export PDF

  1. Από το αριστερό menu επιλέγουμε Export PDF.
  2. Ανοίγει αναφορά με όλες τις ανοιχτές εργασίες που μπορεί να δει ο τρέχων χρήστης.
  3. Πατάμε Εκτύπωση / Save as PDF.
  4. Στο παράθυρο εκτύπωσης του browser επιλέγουμε αποθήκευση ως PDF.
Ο admin εξάγει όλες τις ανοιχτές εργασίες. Ο συνεργάτης εξάγει μόνο όσες επιτρέπεται να βλέπει.

13. Τι κάνει το Reset demo

Το κουμπί Reset demo επαναφέρει τα mock δεδομένα της εφαρμογής στην αρχική τους κατάσταση. Χρησιμοποιείται όταν θέλουμε να ξαναδείξουμε την εφαρμογή από την αρχή ή όταν θέλουμε να καθαρίσουμε τις δοκιμές.

14. Συχνά λάθη που πρέπει να αποφεύγονται

  • Να ξεχνά ο admin να δώσει πρόσβαση στον σωστό συνεργάτη.
  • Να προχωρά εργασία χωρίς να έχουν ανέβει βασικά αποδεικτικά στοιχεία.
  • Να γίνεται έγκριση χωρίς έλεγχο των tabs φωτογραφιών, αρχείων και ιστορικού.
  • Να γίνεται Reset demo χωρίς να γνωρίζει ο χρήστης ότι θα χαθούν οι τοπικές δοκιμαστικές αλλαγές.

15. Σύντομη καθημερινή ροή χρήσης

  1. Ο admin δημιουργεί εργασία.
  2. Ο admin επιλέγει ποιοι συνεργάτες μπορούν να τη δουν.
  3. Ο συνεργάτης μπαίνει και την αναλαμβάνει.
  4. Ο συνεργάτης ξεκινά την εργασία.
  5. Ο συνεργάτης ενημερώνει στοιχεία, φωτογραφίες, αρχεία και υλικά.
  6. Ο συνεργάτης τη στέλνει για επικύρωση.
  7. Ο admin την εγκρίνει ή την επιστρέφει για διόρθωση.
