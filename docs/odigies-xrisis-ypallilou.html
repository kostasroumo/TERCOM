TERCOM Field Operations Manual

Αναλυτικές Οδηγίες Χρήσης Εφαρμογής

Το παρόν εγχειρίδιο περιγράφει τη live έκδοση του TERCOM Field Ops Control. Απευθύνεται σε admins και συνεργάτες πεδίου που χρειάζεται να δημιουργούν, να εκτελούν, να επικυρώνουν και να παρακολουθούν εργασίες μέσα από κοινή βάση δεδομένων.

Περιβάλλον Live εφαρμογή με Supabase login, κοινή βάση και κοινά αρχεία
Ρόλοι TERCOM Admin και συνεργάτες πεδίου
Βασικός στόχος Οργάνωση, εκτέλεση, έλεγχος και ιστορικό εργασιών πεδίου
Αυτοψία Λειτουργίες Ινών Συντήρηση / Λοιπές Εργασίες
Σημαντικό Κάθε ενέργεια αποθηκεύεται στη βάση και συνοδεύεται από ιστορικό αλλαγών. Οι φωτογραφίες και τα αρχεία αποθηκεύονται επίσης κεντρικά, ώστε να είναι κοινά για το σύστημα.

1. Τι κάνει η εφαρμογή

Η εφαρμογή χρησιμοποιείται για την πλήρη διαχείριση εργασιών πεδίου. Κάθε εργασία περιλαμβάνει στοιχεία πελάτη, τοποθεσία, pipeline, στάδιο, ανάθεση, προγραμματισμό, φωτογραφίες, έγγραφα, υλικά, άρθρα εργασιών, survey ασφάλειας και πλήρες ιστορικό.

Ο στόχος δεν είναι απλώς να υπάρχει μια λίστα εργασιών, αλλά να υπάρχει καθαρή ροή από τη δημιουργία μέχρι την τελική έγκριση ή την ακύρωση.

2. Σύνδεση στο σύστημα

  1. Ανοίγετε την εφαρμογή.
  2. Συμπληρώνετε το εταιρικό email και τον κωδικό σας.
  3. Πατάτε Σύνδεση.
  4. Μετά τη σύνδεση εμφανίζεται το προσωπικό σας περιβάλλον, ανάλογα με τον ρόλο σας.
Αν έχετε ρόλο συνεργάτη, θα βλέπετε μόνο τις εργασίες που σας έχουν ανατεθεί. Αν έχετε ρόλο admin, θα βλέπετε όλο το σύστημα.

3. Ρόλοι και δικαιώματα

Ρόλος Τι μπορεί να κάνει
TERCOM / Admin Βλέπει όλες τις εργασίες, δημιουργεί νέες, αναθέτει, εγκρίνει, απορρίπτει, διαχειρίζεται ακυρώσεις και αρχειοθετεί εργασίες.
Partner / Συνεργάτης Βλέπει μόνο τις δικές του εργασίες, ενημερώνει στοιχεία εκτέλεσης, ανεβάζει τεκμηρίωση και τις παραδίδει για επικύρωση.
Admin ως εκτελεστής Ο TERCOM μπορεί, αν μια εργασία ανατεθεί στον ίδιο, να τη χειριστεί και επιχειρησιακά όπως ένας partner.
Περιορισμός συνεργατών Ο συνεργάτης δεν βλέπει φίλτρο ανάθεσης και δεν μπορεί να πειράξει εργασίες άλλων χρηστών.

4. Κύριες ενότητες της εφαρμογής

Dashboard Δείχνει pipelines, status cards και admin queues.
Εργασίες Κεντρική λίστα με αναζήτηση και φίλτρα.
Καρτέλα εργασίας Πλήρες workspace με tabs, notes, uploads, materials, history και actions.
Export PDF Αναφορά με τις ανοιχτές εργασίες του τρέχοντος scope.

5. Pipelines του συστήματος

Pipeline Ρόλος στο workflow
Αυτοψία Αρχική επίσκεψη, αποτύπωση και τεκμηρίωση πεδίου.
Λειτουργίες Ινών Η κύρια εκτέλεση των εργασιών ίνας μετά την αυτοψία.
Συντήρηση / Λοιπές Εργασίες Τελική ή διορθωτική φάση με πρόσθετες εργασίες.

Εσωτερικά στάδια Λειτουργιών Ινών

Στάδιο Σημασία
Χωματουργικό Προετοιμασία όδευσης.
Εμφύσηση Διέλευση / εμφύσηση ίνας.
Κατασκευή εντός κτηρίου Εσωτερικές εργασίες δικτύου.
Ενεργοποίηση Χρησιμοποιείται μόνο όπου απαιτείται από τον πάροχο.
Επιμέτρηση με email Τελικό στάδιο πριν από την επικύρωση.

6. Βασικές καταστάσεις εργασίας

Κατάσταση Τι σημαίνει
Μη ανατεθειμένη Η εργασία δημιουργήθηκε αλλά δεν έχει δοθεί ακόμα σε εκτελεστή.
Ανατέθηκε Υπάρχει εκτελεστής αλλά δεν έχει μπει χρονοπαράθυρο ή έναρξη.
Προγραμματισμένη Υπάρχει εκτελεστής και ημερομηνία έναρξης.
Σε εξέλιξη Η εργασία εκτελείται στο πεδίο.
Ολοκληρωμένο με εκκρεμότητα Η εκτέλεση τελείωσε, αλλά λείπει έγγραφο για να πάει σε επικύρωση.
Για επικύρωση Ο partner παρέδωσε την εργασία για έλεγχο από admin.
Ολοκληρωμένη Η εργασία εγκρίθηκε και έκλεισε.
Ακυρωμένη Η ακύρωση εγκρίθηκε από admin.

7. Καθημερινή ροή Admin

1Δημιουργία εργασίας

Πατάτε Νέα εργασία και συμπληρώνετε τίτλο, pipeline, project, SR ID, BID, πελάτη, τηλέφωνα, διεύθυνση, πόλη, πάροχο και σημειώσεις.

2Άμεση ανάθεση

Από το πεδίο Team / Άμεση ανάθεση μπορείτε να αναθέσετε κατευθείαν την εργασία σε συνεργάτη ή στον ίδιο τον TERCOM.

3Παρακολούθηση από dashboard

Παρακολουθείτε κάθε assignee, τις κάρτες κατάστασης, τα αιτήματα ακύρωσης, τις ακυρωμένες και τις αρχειοθετημένες εργασίες.

4Έγκριση ή επιστροφή

Όταν μια εργασία πάει σε Για επικύρωση, ο admin την εγκρίνει ή την επιστρέφει με σχόλιο.

8. Καθημερινή ροή Συνεργάτη

  1. Μπαίνετε στις Εργασίες.
  2. Ανοίγετε τη δική σας ανατεθειμένη εργασία.
  3. Συμπληρώνετε ή ενημερώνετε σημειώσεις συνεργάτη.
  4. Αν χρειάζεται, βάζετε ή αλλάζετε Ημ/νία έναρξης.
  5. Όταν η εργασία είναι προγραμματισμένη, πατάτε Έναρξη εργασίας.
  6. Ανεβάζετε φωτογραφίες, αρχεία, υλικά, άρθρα εργασιών και safety στοιχεία.
  7. Όταν τελειώσετε, πατάτε Αποστολή για επικύρωση.
Αν λείπει απαραίτητο έγγραφο, η εργασία θα πάει σε Ολοκληρωμένο με εκκρεμότητα. Μόλις ανέβει οποιοδήποτε απαιτούμενο αρχείο, μπορεί να γίνει Επανυποβολή για επικύρωση.

9. Επεξήγηση βασικών πεδίων

Πεδίο Χρήση
Project Το έργο ή η ομάδα έργου στην οποία ανήκει η εργασία.
SR ID Κωδικός αιτήματος / ticket.
BID Δεύτερος κωδικός αναφοράς όπου απαιτείται.
Πάροχος Καθορίζει και τη ροή στα stages των Λειτουργιών Ινών.
Team / Άμεση ανάθεση Ο assignee της εργασίας κατά τη δημιουργία.
Ημ/νία έναρξης Το προγραμματισμένο ξεκίνημα της εργασίας.
Ημ/νία λήξης Χρονική σήμανση ολοκλήρωσης ή κλεισίματος.

10. Τι υπάρχει μέσα στην καρτέλα εργασίας

Tab Τι περιέχει
Κύριος Βασικά στοιχεία, ανάθεση, planning και notes.

11. Φωτογραφίες, αρχεία και τεκμηρίωση

  • Οι φωτογραφίες ανεβαίνουν από το tab Φωτογραφίες.
  • Τα έγγραφα ανεβαίνουν από το tab Αρχεία.
  • Μπορείτε να χαρακτηρίσετε το έγγραφο ως Γενικό έγγραφο, Πιστοποιητικό ή Report / Παράδοση.
  • Για να φύγει μια εργασία από Ολοκληρωμένο με εκκρεμότητα, αρκεί να ανεβεί το απαιτούμενο έγγραφο και να γίνει επανυποβολή.

12. Ακυρώσεις

Ο συνεργάτης μπορεί να στείλει Αίτημα ακύρωσης όταν η εργασία είναι σε εξέλιξη. Για το αίτημα χρειάζεται μόνο αιτιολογία. Δεν απαιτούνται υποχρεωτικά φωτογραφίες ή αρχεία.

Μετά:

  • Ο admin βλέπει το αίτημα στο queue Αιτήματα Ακύρωσης.
  • Μπορεί να το εγκρίνει και η εργασία γίνεται ακυρωμένη.
  • Ή να το απορρίψει και η εργασία συνεχίζει κανονικά.

13. Αρχειοθέτηση

Ο admin μπορεί να πατήσει Αρχειοθέτηση εργασίας. Η εργασία τότε:

  • φεύγει από τις ενεργές λίστες και το dashboard,
  • κρατάει όλο το ιστορικό της,
  • εμφανίζεται μόνο στο admin queue Αρχειοθετημένες Εργασίες.
Η αρχειοθέτηση δεν είναι το ίδιο με την ακύρωση. Η ακύρωση είναι μέρος του workflow. Η αρχειοθέτηση είναι διοικητική ενέργεια για να φύγει μια εργασία από το ενεργό περιβάλλον.

14. Export PDF

  1. Από το αριστερό menu πατάτε Export PDF.
  2. Ανοίγει αναφορά με τις ανοιχτές εργασίες του scope που βλέπετε.
  3. Πατάτε Εκτύπωση / Save as PDF.

15. Καλές πρακτικές

  • Βάλτε πάντα σωστό assignee πριν ξεκινήσει η εργασία.
  • Ελέγξτε το pipeline πριν προχωρήσετε σε επόμενο στάδιο.
  • Πριν από επικύρωση, βεβαιωθείτε ότι υπάρχουν τα σωστά αρχεία.
  • Χρησιμοποιείτε τις σημειώσεις admin και partner για καθαρή επικοινωνία.
  • Μην αρχειοθετείτε εργασία που είναι ακόμη ενεργή στο πεδίο.

16. Σύντομο καθημερινό σενάριο

  1. Ο admin δημιουργεί εργασία.
  2. Ο admin αναθέτει σε συνεργάτη ή στον TERCOM.
  3. Ο εκτελεστής βάζει προγραμματισμό και ξεκινά την εργασία.
  4. Ανεβάζει φωτογραφίες, έγγραφα, υλικά και ενημερώνει notes.
  5. Παραδίδει για επικύρωση.
  6. Ο admin εγκρίνει, επιστρέφει ή διαχειρίζεται τυχόν ακύρωση.
